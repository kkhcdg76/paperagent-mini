# Claude Desktop + PaperAgent-Mini MCP Prompting Guide

## MCP 서버 연결 확인

Claude Desktop을 재시작하면 `paperagent-mini` MCP 서버가 자동 연결됩니다.
연결 확인은 Claude Desktop 설정 → MCP Servers 메뉴에서 가능합니다.

---

## 기본 프롬프트 템플릿

### 1. 논문 검색만 요청하기

```
arXiv에서 {주제} 관련 최신 논문 {N}개를 검색해줘. 논문 제목, 저자, 초록을 표로 정리해줘.
```

**예시:**
```
arXiv에서 "retrieval augmented generation" 관련 최신 논문 5개를 검색해줘. 논문 제목, 저자, 초록을 표로 정리해줘.
```

---

### 2. 논문 한 편 깊게 분석하기

```
{arXiv ID} 논문을 다운로드해서 한국어로 깊이 있게 요약해줘.
아래 항목을 반드시 포함해야 해:
- Problem (해결하려는 문제)
- Key idea (핵심 아이디어)
- Method (방법론)
- Experiments (실험 및 결과)
- Limitations (한계점)
- 우리 프로젝트에 적용 가능성
```

**예시:**
```
2501.04227 논문을 다운로드해서 한국어로 깊이 있게 요약해줘.
```

---

### 3. 여러 논문 종합 리뷰 작성하기

```
{주제} 관련 논문 {N}개를 검색해서 전부 읽고, 한국어로 문헌 조사 리뷰를 작성해줘.
반드시 포함할 내용:
1. 전체 연구 동향 요약
2. 논문 비교 표 (마크다운 테이블)
3. 공통 방법론 정리
4. 미해결 과제 / 한계점
5. 우리가 후속으로 시도할 수 있는 프로젝트 아이디어
```

**예시:**
```
"LLM agents for scientific discovery" 관련 논문 3개를 검색해서 전부 읽고, 한국어로 문헌 조사 리뷰를 작성해줘.
```

---

### 4. 구현 가능한 방법론 추출

```
{주제} 논문 {N}개에서 실제 코드로 구현할 수 있는 알고리즘, 수식, 설계 구조를 추출해줘.
수식은 LaTeX로, 알고리즘은 pseudo-code로 표시해줘.
```

---

### 5. 프로토타입 생성까지 전체 파이프라인

```
{주제}에 대해 논문 검색부터 프로토타입 코드 생성까지 전체 파이프라인을 실행해줘.
```

**예시:**
```
"ReAct: Synergizing Reasoning and Acting in Language Models"에 대해 논문 검색부터 프로토타입 코드 생성까지 전체 파이프라인을 실행해줘.
```

## 고급 프롬프팅: 답변 품질 높이기

### 답변 형식 지정

```
모든 답변은 다음 순서로 작성해줘:
1. [결론 요약] - 3줄 이내
2. [상세 분석] - 항목별 구분
3. [실용적 제안] - 우리가 바로 할 수 있는 것
```

### 한글 요약 품질 요구사항

```
논문 요약 시 다음 규칙을 지켜줘:
- 모든 내용은 한글로 작성
- 전문 용어는 한글 옆에 (영어 원문) 병기
- 각 항목은 글머리 기호(##)로 구분
- 각 항목마다 핵심 키워드 Bold 처리
- 복잡한 개념은 비유/예시를 포함하여 설명
```

### 파이프라인 결과물 구조화

```
파이프라인 실행 후 결과는 다음 마크다운 포맷으로 정리해줘:

# 📊 문헌 조사 서평
(전체 리뷰 내용)

---

# 🔧 핵심 방법론
(알고리즘/수식 추출)

---

# 📋 구현 계획서
(단계별 구현 가이드)

---

# 💻 프로토타입 코드 (별도 파일로 저장)
```

## 사용 가능한 MCP 도구 목록

| 도구 | 기능 | 사용 시점 |
|------|------|-----------|
| `search_arxiv` | arXiv 논문 검색 | 주제 키워드로 검색 시작할 때 |
| `read_paper` | PDF 다운로드 및 텍스트 추출 | 특정 논문 읽을 때 |
| `summarize_paper` | 한국어 논문 요약 | 논문 하나씩 요약할 때 |
| `write_literature_review` | 종합 문헌 서평 | 여러 논문 요약 후 종합할 때 |
| `extract_methods` | 구현 가능 방법론 추출 | 수식/알고리즘 뽑아낼 때 |
| `write_implementation_plan` | 구현 계획서 작성 | 프로토타입 개발 계획 세울 때 |
| `generate_prototype` | mock data 기반 코드 생성 | 실제 실행 가능한 코드 생성할 때 |
| `run_full_pipeline` | 전체 파이프라인 자동 실행 | 한 번에 전부 진행할 때 |

## .env 설정 (LLM 프로바이더)

`2nd_folder\.env` 파일을 수정하여 사용할 LLM을 변경할 수 있습니다:

```ini
# Groq (기본)
LLM_PROVIDER=groq
LLM_MODEL=llama-3.3-70b-versatile
GROQ_API_KEY=your_key_here

# OpenAI
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
OPENAI_API_KEY=your_key_here
```
